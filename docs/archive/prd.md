# Project NeuralFlow - 产品需求文档 (PRD)

## 1. 项目概览
**项目名称**: Project NeuralFlow
**平台**: Android (原生)
**技术栈**: Expo, TypeScript, NativeWind, SQLite (op-sqlite), Reanimated
**核心愿景**: 打造一款商业级的双核 AI 客户端，既是高效的日常聊天工具，也是专业的长篇小说写作助手。
**核心差异**: “极致打磨 (Polished UX)” —— 拒绝 MVP 心态，在 Android 上实现 iOS 级的流畅动画、触觉反馈和视觉细节。

## 2. 用户体验 (UX/UI)
**视觉风格**: “现代极简卡片” (参考: `design_system.md`)
- **主题**: 纯白圆角卡片 (`--surface-primary`) 搭配浅灰背景 (`--surface-secondary`) / 深度暗黑模式。
- **布局**: 设置/菜单采用 Inset Grouped List（内嵌分组列表）。标题使用大标题风格。
- **字体**: Roboto。强调高对比度与易读性。
- **🚫 禁忌**: 严禁使用有色背景容器，严禁使用胶囊/药丸指示器。
- **交互细节**:
    - **消息手势**: 长按气泡唤出上下文菜单 (Context Menu: 复制, 分享, 删除等)。
    - **文本操作**: 气泡内支持原生级的一键复制或精细化选区。
    - **媒体**: 图片查看器 (Lightbox) 支持“保存到相册” (需处理 Android 权限)。
    - **反馈机制**: 所有操作 (成功, 错误, 复制) 必须有优雅的 Toast 提示和精细的触觉反馈 (Haptics)。

## 3. 核心架构：双核驱动

### 3.1. 通用聊天引擎 (Core 1)
- **渲染**: 高性能流式 Markdown 渲染器，支持 LaTeX 公式与代码高亮。
- **多模态**: 支持图片/文件上传与预览。
- **性能**: 极速响应，无掉帧。

### 3.2. 无限本地存储 (Core 2)
- **数据库**: 采用 `op-sqlite` 确保持久化性能。
- **列表渲染**: 使用 `FlashList` 支撑 10万+ 条目的流畅滚动。
- **数据一致性**: 零数据丢失风险。

### 3.3. 写作与上下文模式
- **模式切换**: 顶部可切换“聊天模式”与“写作模式”。
- **写作模式特性**:
    - **滑动窗口**: 智能管理上下文长度。
    - **动态召回 (RAG)**: 定期摘要旧文归档至向量库。自动检索前文情节/人物设定注入 System Prompt。
    - **导出**: 支持一键将当前会话导出为 `.txt` 文件至本地 `Downloads` 目录。

## 4. 功能模块

### 4.1. 全能 API 管理 (Omni API Manager)
- **多服务商**: 支持 OpenAI, Anthropic, DeepSeek, Local (Ollama) 等。
- **高级分组**:
    - *日常主力*: 速度快、成本低的模型。
    - *写作专用*: 高上下文、创造力强的模型。
- **精细配置**: 每个分组独立配置模型参数 (Model, Temperature) 和 System Prompt。
- **热切换**: 聊天界面顶部下拉或按钮，支持对话中实时切换 API 分组，无缝衔接。

### 4.2. RAG 知识库 (Knowledge Base)
- **管理入口**: 独立的一级功能 Tab。
- **导入**: 支持 TXT/PDF/MD 导入。
- **处理**: 自动切片与之本地向量化。
- **挂载**: 允许在特定会话中挂载指定的知识库。

### 4.3. 设置与安全
- **主题引擎**: 动态取色，支持自定义背景图。
- **备份**:
    - 数据库全量导出/导入。
    - 支持 WebDAV 云同步。
- **隐私**: 本地优先 (Local-first) 架构，数据掌握在用户手中。

## 5. 技术指标 (Performance Budget)
- **启动时间**: 冷启动 < 1.5秒
- **交互延迟**: 聊天响应 (UI) < 16ms (60fps)
- **滚动性能**: 列表滚动掉帧率 < 1次/秒
- **离线能力**: 除 API 请求外，全功能离线可用。
- **Android 集成**:
    - Document Picker (文件选择)
    - Media Store (图片保存)
    - File System (存取导出的文本)

## 6. 实施阶段规划
1.  **基础设施**: Expo 初始化, 路由配置, NativeWind 集成, 设计系统落地。
2.  **核心数据层**: SQLite Schema 设计 (Messages, Sessions, Vectors), 封装 FlashList 组件。
3.  **聊天 UI**: 实现消息气泡, 流式渲染逻辑,带有手势的输入栏。
4.  **API 层**: 统一 API 客户端接口, 多服务商与分组逻辑实现。
5.  **高级特性**: RAG 引擎集成, 写作模式逻辑, 文件系统导出。
6.  **打磨交付**: 动画交互优化, 触觉反馈调优, 主题适配, Release 包构建。
