## v1.2.53 详细变更

### 一、知识图谱系统修复

#### 1.1 边去重机制
- **文件**: `src/lib/rag/graph-store.ts`
- **问题**: `createEdge` 方法没有检查是否已存在相同的边
- **修复**: 添加边去重逻辑，相同边累加权重而非重复创建

```typescript
// 检查是否已存在相同的边
const existing = await db.execute(
  `SELECT id, weight FROM kg_edges 
   WHERE source_id = ? AND target_id = ? AND relation = ?
   AND (doc_id = ? OR (doc_id IS NULL AND ? IS NULL))`,
  [sourceId, targetId, relation, docId || null, docId || null]
);

if (existing.rows && existing.rows.length > 0) {
  // 已存在，累加权重
  const newWeight = (existingEdge.weight || 1) + weight;
  await db.execute('UPDATE kg_edges SET weight = ? WHERE id = ?', [newWeight, existingEdge.id]);
  return existingEdge.id;
}
```

#### 1.2 并发竞态条件修复
- **文件**: `src/lib/rag/graph-store.ts`
- **问题**: `upsertNode` 使用"先查询后插入"模式，存在竞态条件
- **修复**: 使用 `INSERT OR IGNORE` 模式避免竞态

```typescript
const insertResult = await db.execute(
  `INSERT OR IGNORE INTO kg_nodes (id, name, type, metadata, created_at, session_id, agent_id) 
   VALUES (?, ?, ?, ?, ?, ?, ?)`,
  [id, name, type, JSON.stringify(metadata), createdAt, scope?.sessionId || null, scope?.agentId || null]
);

if (insertResult.rowsAffected > 0) {
  return id; // 插入成功
}
// 插入失败（节点已存在），执行更新
```

#### 1.3 节点合并自环/重复边清理
- **文件**: `src/lib/rag/graph-store.ts`
- **问题**: `mergeNodes` 可能产生自环和重复边
- **修复**: 合并后清理自环边和重复边

```typescript
// 清理自环边（source_id = target_id）
await db.execute('DELETE FROM kg_edges WHERE source_id = ? AND target_id = ?', [targetId, targetId]);

// 清理重复边（保留权重最高的）
await db.execute(`DELETE FROM kg_edges WHERE id IN (...)`);
```
