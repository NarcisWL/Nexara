# Markdown 预处理器设计与排障手册

> **创建日期**: 2026-02-11
> **关联文件**: `src/lib/markdown/markdown-utils.ts`
> **状态**: 生产中 (Production)

## 背景

不同 LLM 厂商的 Markdown 输出质量差异极大：
- **Gemini** / **Kimi**: 输出规范，标题、列表、分隔符均有完整换行
- **DeepSeek** / **Qwen**: 经常输出"文字墙"——标题无空格(`###标题`)、列表粘连粗体(`***item**:`)、完全零换行

## 核心设计原则

### 1. 通行性 (Universal)
不针对任何特定厂商，所有规则对所有模型输出均适用。

### 2. 幂等性 (Idempotent)
对已正确排版的内容运行后结果不变。这通过 **精准守卫条件** 实现：
- `([^\n#])` — 仅在前方不是换行符 **且** 不是 `#` 时匹配
- `(?!\*)` — 负向前瞻，避免误触合法语法

### 3. 保守性 (Conservative)
仅修复最明确的结构缺陷，不触碰歧义场景。

## 处理流水线

```
输入文本
  │
  ▼
1. LaTeX 分隔符转换 (\[ \] → $$ $$)
  │
  ▼
2. 代码块保护 (```...``` → 占位符)
  │
  ▼
3. 结构化间距修复 (7 条幂等正则)
  │  ├── 3a.  标题（有空格）前确保 \n\n
  │  ├── 3a'. 标题（无空格）修复 + 注入空格
  │  ├── 3a". 行首标题无空格注入
  │  ├── 3b.  分隔符 --- 前后确保 \n\n
  │  ├── 3c.  有序列表前确保 \n
  │  ├── 3d'. 粘连 bullet+bold 拆分 (***x** → * **x**)
  │  └── 3d.  无序列表前确保 \n
  │
  ▼
4. 恢复代码块
  │
  ▼
输出文本
```

## 已知陷阱与排障清单

### 陷阱 1: `#{1,6}` 内部 `#` 自匹配
- **症状**: `### 标题` 被拆碎为 `#\n\n## 标题`
- **根因**: `([^\n])` 匹配了 `###` 中的首个 `#`
- **修复**: 守卫改为 `([^\n#])`

### 陷阱 2: `\s` 匹配换行符
- **症状**: `---\n` 被拆成 `--\n-`; `### 1. Title` 被拆碎
- **根因**: `[-*]\s` 中的 `\s` 匹配了 `\n`，将 `-\n` 当作列表标记
- **修复**: 将 `\s` 改为字面空格 ` `

### 陷阱 3: 粗体闭合 `**` 触发列表
- **症状**: `:**\n* text` 中的 `*` 被当作列表标记的一部分
- **根因**: `([^\n])` 匹配了 `**` 中的 `*`
- **修复**: 守卫改为 `([^\n*])`

### 陷阱 4: 标题无空格 (`###CJK`)
- **症状**: 标题不被识别为标题，渲染为普通粗体文本
- **根因**: `#{1,6}\s` 要求空格，但模型输出无空格
- **修复**: 新增规则 3a'，同时注入 `\n\n` 和空格

## 未修复的已知局限

1. **粗体标签紧跟标题**: `### 标题**Label:**` 中的 `**Label**` 会成为标题的一部分
   - **原因**: 无法安全区分内联粗体和段落标题
   - **影响**: 视觉上标题内容过长，但不影响结构正确性
