# NeuralFlow 项目开发准则与架构规范

## 1. 设置页面架构规范 (Settings Architecture)

为了保持界面整洁并降低认知负荷，设置页面严格划分为两个核心页签，功能归类原则如下：

### **App 标签 (应用个性化)**
- **目的**：处理所有与用户交互、视觉表现和本地应用行为相关的设置。
- **包含内容**：
    - **语言选择**：多语言切换 (i18n)。
    - **外观设置**：主题切换 (Light/Dark/System)、字体大小、色彩方案。
    - **通知管理**：消息提醒开关、声音、震动反馈。
    - **UI 微调**：界面密度、动画开关等。

---

### **Providers 标签 (服务商与数据)**
- **目的**：处理所有与后端模型服务、API 协议及数据持久化相关的设置。
- **包含内容**：
    - **模型服务商配置**：OpenAI/DeepSeek 等 API Keys、Base URL、模型参数 (Temperature/Top-P)。
    - **模型选择**：默认聊天模型、搜索模型、视觉模型。
    - **数据与存储**：本地缓存清理、SQLite 数据库备份/还原、文件存储路径。
    - **安全隐私**：生物识别锁定 (FaceID/Fingerprint)、密钥管理。

---

## 2. 视觉设计原则 (Lumina Aesthetic)

项目采用 **Lumina 设计规范**，区别于原生 iOS 或 Material Design，核心特征如下：

- **背景与层次**：采用纯色背景 (White #FFFFFF / Black #000000)，内容承载于浅灰色 (`bg-gray-50`/`bg-zinc-900`) 的大圆角卡片组件中。
- **图标系统**：严禁使用 iOS 风格的彩色图标块。所有图标必须使用单色线条 (`Monochrome`)，色彩统一使用 `slate-500` 或 `gray-400`。
- **交互反馈**：
    - 关键点击动作必须伴随轻量级震动反馈 (`Haptics.ImpactFeedbackStyle.Light`)。
    - 浮窗采用居中气泡 (`Bubble Menu`) 形式，而非底部动作条 (`Action Sheet`)。

## 3. 代码演进原则
- **上帝组件防御**：单文件超过 500 行必须强制组件化剥离。
- **类型安全**：所有 Mock 数据和 API 返回必须具备完整的 TypeScript 定义。
