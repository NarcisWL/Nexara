### 优先级 4: 未来功能储备 💡
**重要但非紧急**，可在 1.1 版本实现

#### 4.1 写作模式
**价值**: 支持长篇创作场景，如小说、论文  
**核心机制**:
- 自动摘要旧内容（每 10 轮对话）
- 摘要向量化并归档
- 写作时检索前文情节
- 提供"情节回顾"功能

**依赖**: 当前上下文管理已有基础

#### 4.2 PDF 文档支持
**价值**: 扩展知识库输入源  
**技术方案**:
- 使用 `pdf.js` 或 `react-native-pdf` 提取文本
- 集成现有分块与向量化流程

#### 4.3 LaTeX 公式渲染
**价值**: 支持学术与技术内容  
**技术方案**:
- 集成 `react-native-mathjax` 或 `katex`
- 在 Markdown 渲染中识别 `$...$` 和 `$$...$$`

#### 4.4 图片压缩与缓存优化
**价值**: 减少存储占用，提升加载速度  
**技术方案**:
- 上传前自动压缩至 1MB 以下
- 实现 LRU 缓存策略
- 提供"清理缓存"功能

#### 4.5 语音输入
**价值**: 提升输入效率，尤其是移动场景  
**技术方案**:
- 集成 `expo-speech-recognition`（如果可用）
- 或使用 OpenAI Whisper API
- 实现"按住说话"交互

## 🔧 开发工具与资源

### 推荐检查清单工具
```bash
# 1. 扫描未翻译的硬编码文本
rg --type tsx --type ts '"[^"]*[\u4e00-\u9fa5]+[^"]*"' src/

# 2. TypeScript 类型检查
npx tsc --noEmit

# 3. 性能分析（Hermes Profile）
npx react-native profile-hermes

# 4. APK 大小分析
npx react-native-bundle-visualizer
```

### 翻译工作流
1. 提取所有硬编码文本到 Excel/Google Sheets
2. 批量翻译（可辅助使用 AI）
3. 导入到 `zh.ts` 和 `en.ts`
4. 逐个替换组件中的硬编码文本为 `t.xxx.xxx`

### 测试设备建议
- **最低配置**: Android 12, 4GB RAM, 720p 屏幕
- **推荐配置**: Android 13+, 6GB RAM, 1080p 屏幕
- **高端测试**: Android 14, 8GB RAM, 2K 屏幕（验证性能上限）

## 📝 备注

### 技术亮点（营销用）
1. **本地 RAG 引擎**: 无需联网即可向量检索
2. **超级助手**: 全局知识中枢，跨会话检索
3. **极致性能**: 倒序渲染，秒开长对话
4. **隐私优先**: 所有数据本地存储，支持 WebDAV 自托管备份
5. **可定制性**: GIF 头像、自定义 FAB、多主题

### 已知限制（未来改进）
- 仅支持 TXT/MD 文件（PDF 需 1.1 版本）
- 无 LaTeX 公式渲染
- 无语音输入（可能在 1.2 版本）
- 向量检索未使用 ANN 算法（大规模数据库时可能慢）
