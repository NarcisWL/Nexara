# 代码审查检查清单

## 使用方法
复制本清单，在每次 PR/Code Review 时逐项检查。

---

## 通用代码质量

### 架构与组织
- [ ] 单个文件不超过 500 行
- [ ] 复杂逻辑已抽离到独立 hook 或 util
- [ ] 组件职责单一，无"上帝组件"
- [ ] 文件夹结构符合项目约定

### 代码风格
- [ ] 使用 TypeScript 类型注解
- [ ] 无 `any` 类型滥用
- [ ] 变量/函数命名清晰、语义化
- [ ] 注释简洁有用，无冗余注释

### 错误处理
- [ ] 异步操作有 try-catch
- [ ] 用户友好的错误提示
- [ ] 边界情况已考虑（null、undefined、空数组等）

---

## 原生桥接检查（⚠️ 关键）

### Haptics 调用
- [ ] 所有 `Haptics.` 调用在 `setTimeout` 中
- [ ] 延迟时间 >= 10ms
- [ ] 与状态变更同时发生的已延迟

### 状态变更场景
- [ ] `setState` / `zustand.set()` 附近无同步原生调用
- [ ] 语言切换使用延迟模式
- [ ] 主题切换检查是否触发导航重挂载
- [ ] 导航操作附近无同步原生调用

### 嵌套交互
- [ ] 嵌套 TouchableOpacity 使用延迟
- [ ] ScrollView/FlatList 内部点击元素已检查
- [ ] Modal 打开/关闭使用延迟

### 导航器场景
- [ ] Tab 切换使用延迟
- [ ] 任何触发 `key` 变化的操作使用延迟
- [ ] 返回操作附近无同步原生调用

---

## 性能优化

### 渲染优化
- [ ] 列表使用 `React.memo` / `useMemo`
