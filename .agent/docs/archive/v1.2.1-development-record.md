# 开发经验归档: MCP 协议重构与编译流水线优化 (v1.2.1)

> **日期**: 2026-01-30
> **版本**: 1.2.1

## 1. 核心技术突破: MCP Schema 递归转换

### 问题背景
部分 MCP 服务器（如 Alpha Vantage）提供的工具定义极其复杂，包含深层嵌套的对象和数组。
此前的 `McpBridge` 在处理 `jsonSchemaToZod` 时使用了硬编码的非递归逻辑，且大量使用 `z.any()` 逃避验证。
**后果**: 导致模型在调用 `TOOL_CALL` 时看不见参数定义（Schema 为空），只能猜测发送请求，从而引发服务端 500 错误。

### 解决方案
在 `src/lib/mcp/mcp-bridge.ts` 中实现了一套完整的、具备递归能力的 **JSON Schema to Zod** 转换引擎。
- **递归处理**: 自动解析 `properties` (Object), `items` (Array), `enum` (Enum) 等嵌套结构。
- **参数解包**: 让模型能像调用本地工具一样清晰地看到每一个参数及其描述。
- **防御性初始化**: 在 `McpClient.callTool` 中强制确保 `arguments` 至少是一个空对象 `{}`，防止某些服务端因字段缺失而空指针崩溃。

### 经验教训
> [!TIP]
> 永远不要相信 AI 模型对缺失 Schema 的“直觉”。在元协议（Meta-Protocol）层，严格的类型定义是通信稳定性的唯一保障。

## 2. 编译流水线排坑: Worktree 环境适配

### 问题 1: Gradle 代理卡死 (0% Initializing)
**现象**: 在隔离的 Git Worktree 中执行 `./build-release.sh` 时，Gradle 初始化由于连接 Google/Maven 仓库超时而无限卡死。
**根因**: 终端环境变量 `http_proxy` 对独立运行的 Gradle Daemon 并不总是有效，尤其是在复杂的网络转发环境下。
**对策**: 显式向 `android/gradle.properties` 注入 `systemProp.http.proxyHost=127.0.0.1` 及其端口。

### 问题 2: Metro 打包路径冲突 (Unable to resolve module ./index.js)
**现象**: 编译在 `:app:createBundleReleaseJsAndAssets` 阶段失败，报错无法解析入口文件。
**根因**: Git Worktree 具有独特的路径映射逻辑。Expo 生成的原生缓存可能包含过时的物理路径引用，导致打包器在相对路径中走丢。
**对策**: 执行 `npx expo prebuild --platform android --clean` 进行彻底的工程重置，确保入口点与当前物理路径完全同步。

## 3. UI 深度打磨: 暗黑模式下的文字对比度

### 场景
MCP 设置面板中的“已发现工具”小标题。
### 改进
由原始的 `opacity-60` 改为使用 `themeColors.textSecondary` 显式着色。
**理由**: 纯透明度方案在背景色深浅不一时（如 `zinc-900` vs `black`）对比度极不稳定。使用设计的语义颜色（Semantic Color）能保证一致的可读性。
